# AI 工作台初始化页方案

## 1. 页面定位

点击左侧导航进入 AI 工作台时，首页不应该只是一个“去生成”的空白页。

它更适合承担 3 个作用：

1. 告诉用户这个工作台是干什么的
2. 告诉用户第一次应该准备什么
3. 告诉用户如果已经生成过，下一步该继续哪个工作台

所以它本质上应该是：

- 一个轻量首页
- 一个工作台入口页
- 一个最近工作区继续页

而不是单纯空态。

## 2. 初始化页应该展示什么

建议按这 4 层展示。

### 2.1 顶部定位区

展示：

- AI 工作台名称
- 一句话能力说明
- 主 CTA：新增需求

一句话建议：

> 输入需求并生成结构化测试用例，继续完成筛选、补充、执行与回流。

### 2.2 首次使用引导区

如果用户还没有任何生成记录，不要直接显示“暂无记录”，而是展示：

- 先准备什么
- 生成后看什么
- 建议工作流

其中最关键的引导信息是：

#### 先准备什么

- 需求描述 / PRD
- 接口文档或截图
- 代码变更和缺陷上下文

#### 生成后看什么

- 结构化用例列表
- 高风险和待补充项
- 来源、覆盖和执行状态

#### 建议工作流

- 先生成首版用例
- 筛选高风险项补充
- 发布并进入执行回流

### 2.3 最近工作台透出区

如果已经有历史记录，首页建议只透出“最近 3 条”或“推荐继续的 3 条”，不要默认把完整历史作为首屏核心。

建议每条只展示：

- 工作台名称
- 最近更新时间
- 用例数
- 完成率

这是最值得继续使用的最小信息集。

不建议首页首屏透出：

- 全量需求内容
- 全量步骤
- 太多执行明细
- 太重的覆盖分析

这些都应该留给详情页。

### 2.4 完整历史列表

完整历史记录可以继续保留在首页下半部分，但不应该抢占首屏主视觉。

完整列表里建议展示：

- 名称
- 更新时间
- 模块数
- 用例数
- 优先级分布
- 同步状态
- 进度 / 完成率

## 3. 首页要不要展示“已经生成过的内容”

建议展示，但要“摘要展示”，不要“详情透出”。

### 建议展示

- 最近 3 条工作台
- 每条的用例规模
- 完成率
- 更新时间

### 不建议展示

- 某条工作台的完整用例明细
- 复杂表格
- 长文本需求描述
- 执行日志

原因很简单：首页任务是“帮助进入下一步”，不是“完成深度审阅”。

## 4. 首屏推荐信息层级

建议优先级如下：

1. 我能在这里做什么
2. 我第一次要准备什么
3. 我如果做过了，应该继续哪个工作台
4. 我再往下看全部历史记录

## 5. 和列表化方向的关系

既然产品已经明确走“结构化用例列表”路线，那么初始化页的文案和预期也要同步调整：

- 不再强调“生成脑图”
- 改为强调“生成结构化用例”
- 改为强调“后续可筛选、补充、执行、回流”

这样首页的用户心智才会和后续结果页一致。

## 6. 结论

初始化页最适合展示的不是“空”，也不是“全量历史”，而是：

- 一段清晰定位
- 三张轻量引导卡
- 最近 3 条可继续的工作台
- 下方完整历史列表

这样既能服务首次用户，也能服务回访用户。
