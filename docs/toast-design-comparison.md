# Toast 提示设计对比（可视化）

## 当前实现 vs 优化方案

### 问题场景：用例名称很长

假设用例名称为：`测试用户登录功能验证流程包含多因素认证和会话管理的完整测试场景`

---

## 🔴 当前实现（有问题）

```
┌───────────────────────────────────────────────────┐
│ ✓ 用例 "测试用户登录功能验证流程包含多因素认证和 │
│   会话管理的完整测试场景" 已开始执行              │
│                                                   │
│ 点击下方按钮查看 Jenkins 执行详情                 │
│                                                   │
│                              [查看 Jenkins] ──→   │
└───────────────────────────────────────────────────┘
```

**问题**：
- ❌ 用例名称换行，占用大量空间
- ❌ 视觉效果不佳，信息层级混乱
- ❌ 只有一个外部链接，缺少平台内部入口

---

## ✅ 方案 1：简洁型（已实现）

```
┌───────────────────────────────────────────────────┐
│ ✓ 测试用例已开始执行                              │
│                                                   │
│ 执行任务已创建，点击下方按钮查看详情              │
│                                                   │
│                              [查看 Jenkins] ──→   │
└───────────────────────────────────────────────────┘
```

**优点**：
- ✅ 简洁明了，不会换行
- ✅ 信息层级清晰
- ✅ 快速实现，无需修改组件

**缺点**：
- ⚠️ 缺少用例名称信息
- ⚠️ 只有一个操作按钮

---

## 🌟 方案 2：截断型

```
┌───────────────────────────────────────────────────┐
│ ✓ 已开始执行：测试用户登录功能验证流程…           │
│                                                   │
│ 执行任务已创建，点击查看详情                      │
│                                                   │
│                [查看记录]    [查看 Jenkins] ──→   │
└───────────────────────────────────────────────────┘
```

**优点**：
- ✅ 保留部分用例名称，提供上下文
- ✅ 通过截断避免换行
- ✅ 两个操作按钮，提供更多选择

**缺点**：
- ⚠️ 截断后的名称可能不完整
- ⚠️ 需要确定合适的截断长度

---

## 🔥 方案 3：双按钮型（推荐）

```
┌───────────────────────────────────────────────────┐
│ ✓ 测试用例已开始执行                              │
│                                                   │
│ 执行任务已创建，可通过以下方式查看进度            │
│                                                   │
│                [查看记录]    [查看 Jenkins] ──→   │
└───────────────────────────────────────────────────┘
```

**优点**：
- ✅ 简洁明了，不会换行
- ✅ 提供两个操作入口（平台内部 + 外部）
- ✅ 信息清晰，操作明确

**实现方式**：使用自定义 Toast 组件

---

## 🎨 方案 4：悬浮卡片型（高级）

```
┌─────────────────────────────────────────────────────────┐
│ ✓ 测试用例已开始执行                                    │
│ ─────────────────────────────────────────────────────── │
│ 用例名称：测试用户登录功能验证流程…                     │
│ 执行ID：#12345  |  预计耗时：约 2-3 分钟                │
│                                                         │
│ [查看实时日志]    [查看记录]    [查看 Jenkins] ──→     │
└─────────────────────────────────────────────────────────┘
```

**优点**：
- ✅ 提供完整的上下文信息
- ✅ 支持多个操作按钮
- ✅ 可以显示额外信息（预计耗时等）

**缺点**：
- ❌ 占用空间较大
- ❌ 实现复杂度高
- ❌ 可能干扰用户操作

```
│                             │
```

**移动端优化**：

- 文字自动换行，保持可读性
- 宽度自适应屏幕

## 交互流程对比

### 当前实现

```
用户点击"运行"按钮
    ↓
显示 Toast：包含完整用例名称（可能换行）
    ↓
用户点击"查看 Jenkins"
    ↓
跳转到 Jenkins（外部）
```

**问题**：
- 缺少平台内部查看入口
- 用户需要返回平台手动查找执行记录

---

### 优化后（方案 3）

```
用户点击"运行"按钮
    ↓
显示 Toast：简洁提示 + 两个按钮
    ↓
用户选择：
    ├─ 点击"查看记录" → 跳转到平台执行记录页（内部）
    │                   ↓
    │                  查看详细结果、日志、统计信息
    │
    └─ 点击"查看 Jenkins" → 跳转到 Jenkins（外部）
                           ↓
                          查看 Jenkins 构建详情、控制台输出
```

**优势**：
- 提供两个操作路径，用户可自由选择
- 平台内部查看更便捷，无需离开系统
- Jenkins 查看适合需要深入调试的场景

---

## 不同场景下的最佳方案

### 场景 1：用户执行单个用例
**推荐**：方案 1 或方案 3

**理由**：
- 用户明确知道自己点击了哪个用例
- 不需要在 Toast 中重复显示用例名称
- 提供快速跳转链接即可

---

### 场景 2：用户批量执行多个用例
**推荐**：方案 3 或方案 4

**理由**：
- 需要提供更多上下文信息（执行了几个用例）
- 需要提供批次执行记录的链接
- 可以显示预计耗时

**Toast 示例**：
```
┌───────────────────────────────────────────────────┐
│ ✓ 批量执行已开始                                  │
│                                                   │
│ 已提交 5 个测试用例，预计耗时 10-15 分钟          │
│                                                   │
│                [查看记录]    [查看 Jenkins] ──→   │
└───────────────────────────────────────────────────┘
```

---

### 场景 3：用户从任务调度触发执行
**推荐**：方案 3

**理由**：
- 任务可能包含多个用例
- 需要提供任务执行记录的链接
- 需要区分手动执行和自动执行

**Toast 示例**：
```
┌───────────────────────────────────────────────────┐
│ ✓ 任务 "每日回归测试" 已开始执行                  │
│                                                   │
│ 包含 12 个测试用例，预计耗时 30 分钟              │
│                                                   │
│                [查看记录]    [查看 Jenkins] ──→   │
└───────────────────────────────────────────────────┘
```

## 颜色和图标设计

```
┌───────────────────────────────────────────────────┐
│ ✓ 测试用例已开始执行            [绿色图标]        │
│                                                   │
│ 执行任务已创建，可通过以下方式查看进度            │

│  📊 [查看记录]    🔗 [查看 Jenkins] ──→          │

```

- 图标：✓ (check)
- 主色：绿色 (#10b981)
- 背景：浅绿色 (#f0fdf4)

---

### 失败状态（红色）

```
┌───────────────────────────────────────────────────┐
│ ✗ 执行失败                      [红色图标]        │
│                                                   │
│ 请检查 Jenkins 连接或稍后重试                     │
│                                                   │
│  🔄 [重试]        📝 [查看日志] ──→              │
└───────────────────────────────────────────────────┘
```

- 图标：✗ (x)
- 主色：红色 (#ef4444)
- 背景：浅红色 (#fef2f2)

---

### 警告状态（黄色）

```
┌───────────────────────────────────────────────────┐

│ 执行时间超过预期，可能存在问题                    │

└───────────────────────────────────────────────────┘
```

- 图标：⚠ (alert-triangle)
- 主色：黄色 (#f59e0b)
- 背景：浅黄色 (#fffbeb)

## 动画效果建议

### 1. 进入动画
```
Toast 从右侧滑入（slide-in-right）
持续时间：300ms
缓动函数：ease-out
```

### 2. 退出动画
```
Toast 向右侧滑出（slide-out-right）
持续时间：200ms
缓动函数：ease-in
```

### 3. 按钮悬停效果
```
背景色变深
持续时间：150ms
缓动函数：ease-in-out
```
