## API Design Principles（API 设计能力）

### 2.1 能力定义

API Design Principles 指的是 **面向前后端解耦与系统演进的接口抽象与语义设计能力**，其核心目标是：

> 让接口表达业务含义，而不是实现细节，
> 并为未来功能扩展保留结构空间。

---

### 2.2 设计原则（Design Principles）

* **语义清晰优于接口数量少**

  * 一个接口只解决一个业务问题

* **复杂度下沉至服务端**

  * 时间口径（T-1）
  * 聚合计算（成功率、失败数）

* **前端不感知数据来源差异**

  * Jenkins / DB / 缓存对前端透明

---

### 2.3 关键设计规则（Rules & Practices）

* 使用资源化 URL，而非动作式命名

  * `/api/metrics/stability/trend`
* 区分聚合接口与明细接口

  * 聚合接口不返回原始数据
* 时间周期统一使用枚举值（7d / 30d / 90d）

---

### 2.4 项目落地示例

* 稳定性趋势接口：

  * 输入：周期类型、项目维度
  * 输出：日期数组 + 聚合指标
* 失败用例下钻接口：

  * 按日期 + 任务维度返回失败明细
