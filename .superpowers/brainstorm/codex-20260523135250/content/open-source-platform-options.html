完整开源平台接入方案对比

点击你更倾向的方向。这里的判断重点是：完整 UI、自托管、接口自动化、定时执行、CI 触发、报告回收、二次集成成本。

A. MeterSphere 优先

开源企业级持续测试平台，覆盖接口测试、测试计划、定时执行、报告、DevOps 集成。

完整平台 中文生态 REST API 定时任务 GitLab CI 可接
推荐理由

最贴近你的中台接入目标：先部署独立平台，再通过 API/报告链接/回调轮询接入现有平台。

B. RunnerGo 备选 POC

偏接口自动化/性能测试的一体化工具，值得验证部署、API 能力和企业集成细节。

自托管 接口测试 性能测试 需 POC
适合场景

如果更看重轻量和性能压测组合，可以并行做小 POC，但不建议先押主线。

C. Hoppscotch/Bruno 类

更像开源 API 客户端或 Git 友好集合工具，适合开发调试，不太适合作为完整自动化平台核心。

API 客户端 集合管理 CLI 平台能力较弱
不推荐主线

需要你当前平台补大量调度、报告、权限和运行历史能力，会偏离“直接使用完整开源平台”。
