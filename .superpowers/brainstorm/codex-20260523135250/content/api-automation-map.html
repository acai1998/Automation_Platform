接口自动化接入位置

基于当前项目已有的 Jenkins 执行、任务调度、回调和报告链路，接口自动化可以作为新的执行能力接入，而不是重做整个平台。

现有平台

用例管理当前已有 API 用例页入口和测试用例表
任务调度支持 cron、手动、队列、重试、超时释放
执行记录Auto_TestRun / Auto_TestRunResults
报告与看板已有执行状态、统计、WebSocket 进度

建议新增适配层

接口集合同步Postman/OpenAPI/开源工具项目导入
环境与密钥映射变量、域名、认证、敏感值隔离
Runner 适配器统一触发 Jenkins、本地容器或外部 CLI
结果标准化把断言、请求、响应、耗时转成平台结果

开源执行工具

集合编辑/导入类似 Postman/Apifox 的开源方案
命令行执行适合定时、CI/CD、GitLab Webhook
测试报告产物HTML/JUnit/JSON，再回传平台
后续 GitLab 触发代码变更触发接口回归任务

第一阶段重点

先把接口自动化“跑起来、可定时、结果可回收”，尽量复用当前调度与 Jenkins 链路。

后续扩展点

GitLab Webhook 或 CI Pipeline 只需要调用统一任务触发 API，不必关心底层 runner 是什么。
